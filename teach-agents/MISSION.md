# Mission: Understanding AI Agents (with EnM as the proving ground)

## Why
You want genuine, deep literacy in what AI agents *are* and how they work — enough that
when you look at the EnM (Energy & Materials) multi-agent proposals, you can reason about
how each business outcome decomposes into orchestrated agents, what's hard, and what an
implementation actually involves. Agent literacy first; EnM is the motivating, recurring
example that keeps every lesson concrete.

## Success looks like
- You can explain, from memory, the **agent loop** (perceive → reason → act → observe → repeat) and why a "tool-using LLM in a loop" is the core idea.
- You have **built and run a first agent yourself** — a mini "anomaly detector" — for free, end to end.
- You can read any row of `EnM Agents.xlsx` and **sketch the orchestration**: which sub-agents, what each does, how they hand off, where the hard parts are.
- You can tell the difference between what an LLM agent *should* do vs. what belongs in classical code/simulation (a key EnM nuance — many of those agents are NOT just LLM calls).

## Constraints
- Comfortable in Python — lessons use real Python, not pseudocode.
- "Free for learning" — first agent runs on a free LLM tier (Google Gemini free API recommended) or local model; no paid spend required.
- Learning happens over multiple short sessions; each lesson is one tangible win.

## Out of scope (for now)
- Production deployment, scaling, MLOps, security hardening.
- Training/fine-tuning models from scratch.
- Committing to *building* the full EnM systems — right now the goal is to *understand* them, not ship them.
